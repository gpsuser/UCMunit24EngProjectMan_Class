### Question 3

*Adapted from: Question paper January 2024 (Unit 24), Q2(b)(ii); Question paper June 2023 (Unit 24), Q3(e)*

**Case study context:** Priya is the project manager at Wrenfield Engineering, a structural steel fabricator based in Sheffield. Wrenfield is implementing a new automated welding system to reduce production cycle times. The project is now at the end of Month 3 of a planned twelve-month programme. Priya has been fully occupied managing supplier relationships and has not been formally comparing actual production milestones against the project schedule.

**(a)** Identify **one** control technique that Priya could use to help keep the automated welding project on track.

[1 mark]

**(b)** Explain **two** consequences to Wrenfield Engineering of Priya failing to compare actual progress with the intended project schedule.

[2 marks]

### Question 5

**Case study context:** Callum is a project manager at Pennine Infrastructure Ltd, which is installing a new water treatment facility. His line manager has asked him to introduce regular project status reporting.

**(a)** State **two** purposes of a project status report.

[2 marks]

**(b)** Callum uses a RAG status system in his reports. The current report shows an overall status of AMBER for the schedule and RED for cost.

Explain what these RAG statuses indicate to the project board about the current state of the project.

[2 marks]

---

### Question 6

**Case study context:** Sienna is a project manager at Castlefield Rail, a company delivering the refurbishment of a signal control room for a major UK train operating company. The project plan included a tolerance of ±8% on the overall budget and ±5 working days on the programme.

At the end of Week 14, the project is running 12 days behind schedule. The cost of materials for Phase 2 has increased by 11% above budget due to supply chain delays.

**(a)** Explain why Sienna should escalate these issues rather than attempt to resolve them independently at project team level.

[2 marks]

**(b)** Describe the **three** stages Sienna should follow when formally escalating an issue through the project governance structure.

[3 marks]

---

### Question 7

**Case study context:** Hargreaves Fabrications is delivering a large-scale structural steelwork contract for a new distribution warehouse. Midway through the project, the client requests several modifications: adding extra mezzanine floor supports, changing the roofline profile, and substituting a cheaper steel grade for non-structural cladding supports. The project manager, Donna, agrees to the first two changes verbally with the client and makes a note in her personal diary. The third change is submitted on a formal change request form and goes through the company's change control process.

**(a)** Explain the difference between **scope creep** and **managed change**.

[2 marks]

**(b)** Identify which of the three changes described above represents scope creep, and explain why.

[2 marks]

**(c)** Describe **two** steps in a formal change control process that Donna should have followed for all three changes.

[2 marks]
