### Question 1 — Mark Scheme

**Source: Jan 2024 (Unit 24), Q1(b) | LO1.2 — Outline plan**
**Total marks: 6 (3 topics × 2 marks each)**

**Marking guidance:**
Award **1 mark** for correctly naming or identifying each topic, and **1 further mark** for a description of what that topic means. Do not award the second mark for an explanation of *why* the topic is important — the question asks for description only.

Acceptable topics and their descriptions (any three):

```
+--------------------+--------------------------------------------------------+
| Topic              | Acceptable description (1 mark each)                  |
+--------------------+--------------------------------------------------------+
| Purpose            | Explains the reason why the project is being           |
|                    | undertaken and what overall aim it is intended to      |
|                    | achieve.                                              |
+--------------------+--------------------------------------------------------+
| Aims               | The broad goals the project intends to reach,          |
|                    | expressed as general statements of intent rather       |
|                    | than specific measurable targets.                     |
+--------------------+--------------------------------------------------------+
| Objectives         | Specific, measurable targets (often SMART) that        |
|                    | define what the project must deliver within a          |
|                    | defined timeframe.                                    |
+--------------------+--------------------------------------------------------+
| Scope              | Defines the boundaries of the project — what is        |
|                    | included in and excluded from the work, to prevent     |
|                    | scope creep.                                          |
+--------------------+--------------------------------------------------------+
| Constraints        | Documents the limitations on the project such as       |
|                    | fixed budget, deadline, regulatory requirements, or    |
|                    | resource availability.                                |
+--------------------+--------------------------------------------------------+
| Deliverables       | The tangible or measurable outputs that the project    |
|                    | must produce by the end of each stage or at           |
|                    | completion.                                           |
+--------------------+--------------------------------------------------------+
| Targets/Goals      | Specific performance benchmarks or milestones the      |
|                    | project must achieve at defined points in time.       |
+--------------------+--------------------------------------------------------+
| Communication      | Outlines how and when project information will be      |
| strategy           | shared among team members, stakeholders, and the       |
|                    | sponsor throughout the project.                       |
+--------------------+--------------------------------------------------------+
| Controls           | Identifies the time, cost, and quality strategies      |
|                    | that will be used to monitor and manage the project   |
|                    | throughout its lifecycle.                             |
+--------------------+--------------------------------------------------------+
```

**Common errors to address in feedback:**
- Providing an explanation of *why* the topic matters rather than a description of *what* it is. Encourage students to read the command word carefully: "describe" requires meaning, not justification.
- Listing the same topic twice under different names (e.g., "aims" and "goals" as two separate answers). Only one award is appropriate for essentially identical content.
- Vague responses such as "a timeline" — this is too imprecise; "controls: the time, cost, and quality strategies used to keep the project on track" would be acceptable.

### Question 2 — Mark Scheme

**Source: Jan 2024 (Unit 24), Q1(c) | LO4.4 — Validity criteria**
**Total marks: 3 (1 mark per correct match)**

**Correct matches:**

```
+---------------------+        +------------------------------------------------+
|     Quality         |------->| The information is relevant to the needs of the |
|                     |        | project and has sufficient detail and accuracy  |
|                     |        | to meet the required standard.                 |
+---------------------+        +------------------------------------------------+

+---------------------+        +------------------------------------------------+
|     Reliability     |------->| The information can be reproduced — if the     |
|                     |        | research were repeated, the same results would  |
|                     |        | be obtained.                                   |
+---------------------+        +------------------------------------------------+

+---------------------+        +------------------------------------------------+
|     Importance      |------->| The information is significant to the business  |
|                     |        | and would affect project decisions if it were   |
|                     |        | found to be incorrect or unavailable.           |
+---------------------+        +------------------------------------------------+
```

**Distractors not matched:**
- "The information is up to date and still applicable to the current situation." → this is **Currency**, not matched.
- "The information comes from a credible and trustworthy source with no reason for distortion." → this is **Integrity of source**, not matched.

**Common errors to address in feedback:**
- Confusing **Reliability** with **Accuracy**. Reliability means reproducibility (same results if the study is repeated); accuracy means the information is correct and free from errors. These are two distinct criteria.
- Confusing **Quality** with **Integrity of source**. Quality refers to the standard of the information meeting the project's needs; integrity of source refers to the trustworthiness of the person or organisation supplying it.
- Selecting "up to date" as the definition of Reliability. Currency (up-to-dateness) is a separate criterion.

### Question 3 — Mark Scheme

**Adapted from: Jan 2024 (Unit 24), Q1(b) | LO1.2 — Outline plan (contextualised)**
**Total marks: 6 (2 topics × 3 marks each)**

**Marking guidance:**
This question adds a further demand compared with Question 1: in addition to identifying and describing the topic (1 mark), students must explain why it is *particularly important* given the specific context of the Apex Structures depot project (2 marks). Award marks as follows:

- **1 mark**: Correct identification and clear description of the topic.
- **2 marks**: A well-contextualised explanation that refers specifically to the depot project scenario (e.g., the 16-month contractual deadline, the £3.5 million budget, the specialist substation, the financial penalty clause).

**Indicative answers (any two of the following):**

```
+--------------+-------------------------------------------------------------+
| Topic        | Full answer (indicative — accept equivalent phrasing)      |
+--------------+-------------------------------------------------------------+
| Constraints  | Constraints are the limitations imposed on the project     |
|              | (1 mark). In the case of this depot project, the           |
|              | contractual 16-month deadline and the £3.5 million budget  |
|              | are both hard constraints — failure to complete within the |
|              | deadline triggers financial penalties payable to the rail  |
|              | operator, so documenting these at the outset is critical   |
|              | for the team to understand the boundaries within which     |
|              | all planning decisions must be made (2 marks).            |
+--------------+-------------------------------------------------------------+
| Scope        | Scope defines what work is included in and excluded from   |
|              | the project (1 mark). For this project, it is essential to |
|              | define clearly whether the electrical substation is within |
|              | scope or is a separate contract, and whether landscaping   |
|              | or rail connection works fall within Apex Structures'      |
|              | responsibility — ambiguity here could lead to scope creep  |
|              | that threatens both the budget and the deadline (2 marks). |
+--------------+-------------------------------------------------------------+
| Objectives   | Objectives are the specific, measurable targets the        |
|              | project must achieve (1 mark). Writing SMART objectives    |
|              | for this project — for example, "complete the structural   |
|              | frame to BS EN 1993 standard by Month 8 within a          |
|              | sub-budget of £850,000" — provides the project manager     |
|              | with clear benchmarks against which to measure progress    |
|              | during the 16-month programme (2 marks).                  |
+--------------+-------------------------------------------------------------+
| Controls     | Controls are the time, cost, and quality strategies used   |
|              | to monitor the project (1 mark). Given the financial       |
|              | penalty clause in this project, establishing cost control  |
|              | procedures from the outset — such as regular budget        |
|              | variance reviews — is particularly important because any   |
|              | uncontrolled cost overrun risks both the project margin    |
|              | and the relationship with the rail operator client         |
|              | (2 marks).                                                |
+--------------+-------------------------------------------------------------+
```

**Teaching note:** The key discriminator in this question compared with Question 1 is contextualisation. Students who write generic descriptions without linking to the depot project scenario, the 16-month deadline, or the penalty clause should be redirected — the examiner consistently awards additional marks for answers that demonstrate applied understanding rather than recalled definitions.

### Question 4 — Mark Scheme

**Adapted from: Jan 2024 (Unit 24), Q1(c) | LO4.4 — Validity criteria (new set)**
**Total marks: 3 (1 mark per correct match)**

**Correct matches:**

```
+---------------------+        +------------------------------------------------+
|     Accuracy        |------->| The information is free from errors and         |
|                     |        | correctly represents the facts as they          |
|                     |        | actually are.                                  |
+---------------------+        +------------------------------------------------+

+---------------------+        +------------------------------------------------+
|     Currency        |------->| The information is up to date and remains       |
|                     |        | applicable to the current project context.      |
+---------------------+        +------------------------------------------------+

+---------------------+        +------------------------------------------------+
|     Bias            |------->| The information has been influenced by          |
|                     |        | personal opinion or a vested interest, which   |
|                     |        | may distort the findings.                      |
+---------------------+        +------------------------------------------------+
```

**Distractors not matched:**
- "The information is significant to the business and would affect decisions if it were unavailable." → this is **Importance**.
- "The information is reproducible — repeating the research process would yield the same results." → this is **Reliability**.

**Common errors to address in feedback:**
- Confusing **Accuracy** with **Reliability**. Accuracy refers to correctness of content; reliability refers to reproducibility of results. Draw students' attention to the fact that it is possible for a study to be reliable (same results each time) but inaccurate (consistently wrong due to a flawed methodology).
- Failing to recognise **Bias** as a specific criterion. Students sometimes describe bias as a research method limitation rather than a validity criterion. Reinforce that bias is one of the ten criteria listed in the Unit Specification (OCR, 2017) and must be defined in its own terms.

### Question 5 — Mark Scheme

**LO2.2 — Leadership styles**
**Total marks: 6 (2 styles × 3 marks each)**

**Marking guidance:**
- **1 mark**: Correct identification of a recognised leadership style (e.g., directive/autocratic, coaching/collaborative, supportive/participative, delegative/laissez-faire).
- **2 marks**: A contextualised explanation that links the style to the specific group described in the question (newly qualified engineers OR senior engineers), explaining why it is appropriate for that group.

**Indicative answers:**

```
+---------------------------+------------------------------------------------+
| Style (for newly          | Explanation (indicative)                       |
| qualified engineers)      |                                                |
+---------------------------+------------------------------------------------+
| Directive / Autocratic    | This style involves the leader providing clear |
|                           | instructions and close supervision with little |
|                           | delegation of decision-making. It is           |
|                           | appropriate for the newly qualified engineers  |
|                           | because they have limited site experience and  |
|                           | require guidance to ensure that work is        |
|                           | completed safely and to the required           |
|                           | specification. Leaving them to work without    |
|                           | direction could lead to quality failures or    |
|                           | safety incidents on a complex live-site        |
|                           | project.                                      |
+---------------------------+------------------------------------------------+
| Coaching / Collaborative  | In a coaching style the leader works closely   |
|                           | with team members, explaining reasoning and    |
|                           | encouraging skill development. For newly       |
|                           | qualified engineers who are willing but        |
|                           | inexperienced, this supports their professional|
|                           | development while ensuring the project manager |
|                           | retains oversight of critical activities.     |
+---------------------------+------------------------------------------------+
```

```
+---------------------------+------------------------------------------------+
| Style (for senior         | Explanation (indicative)                       |
| engineers)                |                                                |
+---------------------------+------------------------------------------------+
| Delegative / Laissez-faire| This style involves the leader delegating       |
|                           | decisions and tasks to team members with        |
|                           | minimal intervention. It is appropriate for    |
|                           | the six senior engineers because they are       |
|                           | described as highly skilled and self-motivated.|
|                           | Micro-managing experienced professionals can   |
|                           | reduce morale and productivity; delegating     |
|                           | responsibility instead allows them to use their|
|                           | expertise efficiently, which also frees        |
|                           | Ravinder to focus on managing the less         |
|                           | experienced members of his team.              |
+---------------------------+------------------------------------------------+
| Supportive /              | A supportive style maintains a collaborative   |
| Participative             | relationship where the leader is available for |
|                           | guidance but does not direct the work in       |
|                           | detail. This suits senior engineers who are    |
|                           | competent and do not require instruction, but  |
|                           | who may still benefit from having the project  |
|                           | manager available to resolve issues that       |
|                           | require cross-team coordination or             |
|                           | escalation.                                   |
+---------------------------+------------------------------------------------+
```

**Teaching note:** The four-quadrant Hersey–Blanchard situational leadership model (Hersey and Blanchard, 1969) is the theoretical basis for this question. Students do not need to name the model to score full marks, but they must demonstrate that their chosen style is matched to the competence and motivation level of the specific group. Students who identify two contrasting styles but provide generic explanations — without referencing the newly qualified or senior engineers specifically — should receive only 1 mark per style.

### Question 6 — Mark Scheme

**LO3.3 — PERT technique**
**Total marks: 3**

**Marking guidance:**

```
Mark | Criterion
-----+-------------------------------------------------------------------
  1  | Correctly recalls and states the PERT formula:
     |   E = (O + 4M + P) / 6
-----+-------------------------------------------------------------------
  1  | Correctly substitutes the given values:
     |   E = (5 + (4 × 8) + 17) / 6
     |   E = (5 + 32 + 17) / 6
     |   E = 54 / 6
-----+-------------------------------------------------------------------
  1  | Correct final answer: E = 9 weeks
```

**Full worked solution:**

```
PERT CALCULATION — Steel Frame Erection
=========================================

Formula:    E = (O + 4M + P) / 6

Given:      O = 5 weeks
            M = 8 weeks
            P = 17 weeks

Step 1:     4 × M = 4 × 8 = 32

Step 2:     O + 4M + P = 5 + 32 + 17 = 54

Step 3:     E = 54 / 6 = 9 weeks

Expected duration (E) = 9 weeks
```

**Common errors to address in feedback:**
- Using the simple arithmetic mean (O + M + P) ÷ 3 = (5 + 8 + 17) ÷ 3 = 10 weeks. This is incorrect because it fails to apply the weighting factor of 4 to the most likely estimate. Stress to students that the factor of 4 reflects the greater confidence placed in the most likely estimate, and that omitting it is the single most common error on PERT questions.
- Arithmetic errors in the multiplication step (4 × M). Encourage students to show working clearly so that the substitution mark can be awarded even if the final arithmetic is wrong.
- Giving the answer without units. Although units are not typically penalised in OCR mark schemes, it is good professional practice and worth reinforcing.

### Question 7 — Mark Scheme

**LO4.5 — Data collation and decision making**
**Total marks: 4**

**Marking guidance:**

```
Mark | Criterion
-----+-------------------------------------------------------------------
  1  | Clear decision stated: the project is NOT meeting its target
     | (accept equivalent wording such as "below target" or
     | "failing to achieve the minimum required average").
-----+-------------------------------------------------------------------
  1  | Correct calculation of the mean actual pour volume:
     |   (115 + 108 + 122 + 110) / 4 = 455 / 4 = 113.75 m3 per week
     |   (or use total actual: 455 m3 over 4 weeks, which is 25 m3
     |    below the 480 m3 planned total — accept either approach)
-----+-------------------------------------------------------------------
  2  | Well-justified evaluation that:
     | — references specific values from the table (e.g., identifies
     |   Week 2 as the worst performing week at -12 m3 variance, or
     |   notes that only Week 3 exceeded the target); AND
     | — identifies a pattern or trend (e.g., three of the four weeks
     |   were below target; the project showed no improvement trend
     |   across the four weeks, suggesting a systemic issue).
     | Award 1 mark for either criterion above; 2 marks for both.
```

**Full model answer:**

The project is **not** meeting its scheduled concrete pour target. (1 mark)

The mean actual pour volume over the four weeks is (115 + 108 + 122 + 110) ÷ 4 = **113.75 m³ per week**, which is below the minimum required average of 115 m³ per week. (1 mark)

Three of the four weeks recorded actual volumes below the 120 m³ planned target, with Week 2 representing the worst performance at only 108 m³ — a shortfall of 12 m³. Although Week 3 marginally exceeded the planned volume (+2 m³), this was insufficient to compensate for the cumulative deficit. The total actual pour volume of 455 m³ is 25 m³ below the planned total of 480 m³, and there is no clear upward trend in the data to suggest that performance is improving. This indicates a systemic issue rather than an isolated anomaly, and the project manager should investigate whether plant capacity, labour availability, or material supply is restricting output before the next stage of work begins. (2 marks)

**Teaching note:** The OCR mark scheme for LO4.5 questions consistently rewards three elements: a clear statement of decision, a calculation to support the decision, and a justified conclusion that references specific data. Students who make a correct decision but provide no calculation or data reference will score a maximum of 1 mark. Reinforce the three-step structure: **Decide → Calculate → Justify with data**.
